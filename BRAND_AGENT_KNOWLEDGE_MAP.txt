CONSTRAINT PROPAGATION ARCHITECTURE
====================================

Brand Agent Output → Downstream Constraints
============================================

TONE MATRIX
├─→ Copy Agent: "Use 70% friendly, 20% technical in product emails"
├─→ Social Agent: "LinkedIn: expert tone, Twitter: friendly tone"
├─→ Support Agent: "Customer emails: 80% empathetic, 15% solution-focused"
└─→ Design Agent: "Conversational UI copy, avoid corporate jargon"

DESIGN TOKENS
├─→ Design Agent: "Button primary: #0066CC (±2 RGB tolerance)"
├─→ Frontend Agent: "Use --color-primary CSS variable, not hardcoded hex"
├─→ Build Agent: "Validate tokens in Figma + CSS before deploy"
└─→ QA Agent: "WCAG compliance: 4.5:1 contrast on all text"

POSITIONING STATEMENT
├─→ Copy Agent: "All marketing copy must reinforce 'only for EU merchants' positioning"
├─→ Sales Agent: "Competitive slide decks must reference Onlyness statement"
├─→ Product Agent: "Feature prioritization aligned with positioning"
└─→ Social Agent: "Thought leadership content must reinforce core positioning"

COMPLIANCE REQUIREMENTS (EU/GDPR)
├─→ Infrastructure Agent: "All data EU-hosted, no US CDNs"
├─→ Analytics Agent: "Use Plausible, not Google Analytics"
├─→ Design Agent: "Self-host fonts, no Google Fonts CDN"
├─→ Legal Agent: "All third-party integrations require DPA"
└─→ Security Agent: "Audit all vendor GDPR compliance quarterly"
