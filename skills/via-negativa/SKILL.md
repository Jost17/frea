# Via Negativa

**Nassim Taleb's Prinzip** - Robustheit durch Subtraktion.

> Statt "Was kann ich hinzufuegen?" → **"Was kann weg?"**

## Der 3-Schritt-Prozess

| Schritt | Aktion | Methode |
|---------|--------|---------|
| 1. INVENTAR | Vollstaendige Liste | Was ist da? |
| 2. PRUEFUNG | "Wuerde ich es heute hinzufuegen?" | Nein → Kandidat |
| 3. STREICHUNG | Mutig entfernen | Im Zweifel: Weg damit |

## Die 3 Kernfragen

1. **"Wuerde ich das heute noch einfuehren?"** → Nein = Streichen
2. **"Was passiert, wenn ich es entferne?"** → Nichts = Sofort streichen
3. **"Rechtfertigt der Nutzen die Komplexitaet?"** → Nein = Streichen

## Anwendungsbereiche

| Bereich | Frage |
|---------|-------|
| **Code** | Dead code, unused imports, ueberfluessige Abstraktionen? |
| **Tools** | Apps/Subscriptions die ich nie nutze? |
| **Prozesse** | Meetings ohne klares Ergebnis? |
| **Features** | Settings die niemand nutzt? |
| **Skills** | Veraltete Skills, Duplikate? |

## Abgrenzung zu Invert

| Aspekt | Via Negativa | Invert |
|--------|--------------|--------|
| Frage | "Was kann weg?" | "Was wuerde scheitern?" |
| Timing | Nach Akkumulation | Vor Aktion |
| Output | Streichliste | Risikolistel |

**Komplementaer:** Invert vor Projekten, Via Negativa fuer bestehende Systeme.
