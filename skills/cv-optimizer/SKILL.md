# CV-Optimizer Skill

## Quick Reference

### Workflows
| Trigger | Workflow |
|---------|----------|
| "optimiere CV fuer Job" | Job-spezifische Optimierung |
| "authenticity check" | BS-Detektor |
| "CV generieren" | CV-Version erstellen |

### Authenticity Dial (Ton)
| Stufe | Ton | Beispiel |
|-------|-----|----------|
| **Faktisch** | Nur Fakten | "QA fuer 14-Maerkte-Rollout durchgefuehrt" |
| **Confident** | + Framing | "Als QA-Lead 14-Maerkte verantwortet" |
| **Bold** | Stark | "14-Maerkte-Rollout mit 0 Bugs" |

**Default:** Confident

## Wichtige Regeln

### Anti-Patterns (NIEMALS verwenden)
- "Results-driven leader"
- "Visionary leader"
- "Synergien"
- "Transformiert"
- "Governance" (statt "Struktur")

### Ton-Regeln
- Kurze Saetze (8-15 Woerter)
- Aktive Stimme
- Zahlen statt Adjektive
- Ergebnis > Taetigkeit
- Understatement > Overstatement

### Stretch-Score
| Score | Bedeutung |
|-------|-----------|
| 1-2 | Fakt/Geframet — OK |
| 3 | Mit Begruendung |
| 4-5 | NIE — ersetzen |

**Durchschnitt max 2.5**

## Bausteine

**Location:** Josts Obsidian Vault unter `cv/bausteine/`

Jeder Baustein = authentische Erfahrung. CV wird aus Bausteinen zusammengesetzt, nicht neu geschrieben.

## Workflows

### 1. Optimize for Job
1. URL sichern (PFLICHT)
2. JD analysieren
3. Bausteine matchen
4. Tailored CV-Sektionen vorschlagen

### 2. Authenticity Check
1. Text entgegennehmen
2. Anti-Patterns laden
3. Jeden Bullet bewerten (Score 1-5)
4. Alternativen vorschlagen

### 3. Generate Version
1. Parameter klaeren (Zielgruppe, Ton)
2. Bausteine zusammensetzen
3. Markdown + docx generieren
4. TELOS-Check
