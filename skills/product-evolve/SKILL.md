# Product Evolve Skill

Systematische Weiterentwicklung: Codebase-Analyse → Interview → Research → Vorschlaege → Routing.

## 6-Phasen Workflow

| Phase | Aktion |
|-------|--------|
| 1. Identifizieren | Projekt-Name + Codebase-Pfad |
| 2. Bestandsaufnahme | Codebase lesen, TELOS-Ziele |
| 3. Interview | 5 Fragen (Erstaufruf) / 1-2 Fragen (Folgeaufruf) |
| 4. Research | 3 parallele Agenten (Best Practices, Tech-Scan, Vault-Scan) |
| 5. Vorschlaege | Quick Wins / Strategisch / Moonshots mit Impact-Effort |
| 6. Routing | 🟢 Things3, 🟡 Obsidian Inbox, 🔴 Rejected |

## Key Principles

- **Context-File:** `context/{projekt-slug}.json`
- **Adaptive:** Klein (<10 Files) → weniger Agenten
- **Report:** `300 Resources/Evolve-Reports/{slug}_{datum}.md`
