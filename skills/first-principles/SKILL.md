# First Principles Thinking

Probleme auf fundamentale Wahrheiten reduzieren, Annahmen eliminieren, von Grund auf neu bauen.

## Wann nutzen?

- Komplexe Probleme, wo Standard-Loesungen nicht passen
- "Weil man das so macht"-Denken durchbrechen
- Optimierung (Kosten, Zeit, Ressourcen)
- Strategische Entscheidungen

## Der 6-Phasen-Prozess

| Phase | Aktion | Output |
|-------|--------|--------|
| 1. STOP & FRAME | Problem klar definieren | Problem-Statement |
| 2. DEKONSTRUIERE | In Grundbestandteile zerlegen | Komponenten-Map |
| 3. ASSUMPTION AUDIT | Alle Annahmen identifizieren + pruefen | Annahmen-Liste |
| 4. FUNDAMENTALS | Unveraenderliche Wahrheiten finden | Fundamental-Liste |
| 5. NEU BAUEN | Loesung von Fundamentals aus konstruieren | Neue Loesung |
| 6. VALIDATE | Gegen urspruengliches Problem pruefen | Validierte Loesung |

## Annahmen-Kategorien (Phase 3)

1. Explizite (bewusst getroffen)
2. Implizite (unbewusst uebernommen)
3. Industrie-Konventionen ("so macht man das")
4. Technische ("geht nicht anders")
5. Ressourcen ("nicht genug X")
