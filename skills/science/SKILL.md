# Science Skill - Hypothesis-Driven Analysis

Falsifizierbare Hypothesen aufstellen, testen und auswerten (Karl Popper).

## Wann nutzen

- Ungepruefte Annahmen im Raum
- Debugging: "Ich glaube der Fehler liegt bei X" → Teste es!
- Performance: Nicht raten, messen
- Confirmation Bias droht

## 5-Phasen-Prozess

| Phase | Ziel | Output |
|-------|------|--------|
| 1. Beobachten | Phaenomen beschreiben (Fakten, keine Interpretation) | Messbare Problembeschreibung |
| 2. Hypothese | Falsifizierbare Aussage: "Wenn [X], dann [Y], weil [Z]" | Hypothese + Widerlegungskriterium |
| 3. Vorhersage | Konkrete, testbare Vorhersagen ableiten | Tabelle: Bestaetigung vs. Widerlegung |
| 4. Test | Experiment durchfuehren (Benchmark, A/B, Backtest) | Testergebnisse mit Zahlen |
| 5. Auswerten | Bestaetigt / Widerlegt / Angepasst | Klare Entscheidung + Begruendung |

## Quick-Start

```
Beobachtung: [Messbare Fakten]
H1: Wenn [Bedingung], dann [Ergebnis], weil [Mechanismus].
Widerlegt wenn: [konkretes Gegenbeispiel]
Test: [Methode] → Ergebnis: [Zahlen]
Urteil: BESTAETIGT / WIDERLEGT / ANGEPASST
```
