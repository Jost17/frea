# arXiv Deep-Dive Skill

Stufe 2 des Research-Workflows: PDF-Analyse → konkrete Aktionen.

## Trigger

- "Deep-Dive auf die Top-Papers"
- "Analysiere Paper 2601.XXXXX"
- "arXiv Deep-Dive"

## Workflow

| Schritt | Aktion |
|---------|--------|
| 1. Paper-Auswahl | Aus Report übernehmen |
| 2. Integration Gate | Capability Audit + Memory Check |
| 3. PDF Download | curl → /tmp/ |
| 4. Extraktion | Security / Agent / Quant |
| 5. Validierung | Umsetzbarkeit, Aufwand |
| 6. Priorisierung | Sofort / Sprint / Backlog |
| 7. Output | Summary + Things3 Task |

## Input-Optionen

| Input | Beschreibung |
|-------|--------------|
| Paper-IDs | "Deep-Dive 2601.01723, 2601.01857" |
| Auto-Detect | Heutiger Report: Security + Score 11+ |

**Maximum:** 5 Papers pro Session

## Integration Gate

Prüft ob Paper Mehrwert bringt:

| Check | Frage |
|-------|-------|
| Capability Audit | Hat Paul II das schon? |
| Memory Check | Wurde schon analysiert? |
| Gap Analysis | Lohnt sich der Aufwand? |

**Empfehlungen:**
- IMPLEMENT → Neuer Bereich
- SKIP → Bereits abgedeckt
- REVIEW → Manuell entscheiden
